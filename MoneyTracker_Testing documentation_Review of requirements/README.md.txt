## Description

This project helped me to gain practical experience in analyzing and reviewing software requirements. I learned how to use review techniques to identify inaccuracies, bugs, ambiguities, and other issues in the requirements.

## Goal

Learn to use free-form review techniques. Apply the knowledge gained to analyze and review the requirements for the mobile app MoneyTracker.

## What was done

Applied freestyle review techniques to analyze specification documents
Identified inconsistencies, ambiguities, missing validations, UX issues, and documentation mismatches.
Reported over 10 defects across multiple sections of the documentation.
